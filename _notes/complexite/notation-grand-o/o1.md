# Complexité O(1) - Constante

Un algorithme en O(1) s’exécute en un temps fixe, peu importe la taille des données. Cela signifie que l’opération ne dépend pas de n. Par exemple, accéder au premier élément d’un tableau ou retourner une valeur calculée sans boucle. Même si le tableau contient 10 ou 10 000 éléments, le temps reste le même.

## Exemple
```java
public class ExempleO1 {
    public static int getFirst(int[] array) {
        return array[0]; // Accès direct, toujours constant
    }
}
```
